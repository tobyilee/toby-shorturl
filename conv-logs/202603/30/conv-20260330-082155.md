## 3. Google OAuth2 로그인 후 404 디버깅
사용: Chrome browser tools, Bash, Edit, Write

Toby: Google 로그인 후 dashboard에서 404 에러 발생.

Claude:
- DB에 유저가 저장되지 않음을 확인 (users 테이블 비어있음)
- 1차 시도: `CustomOAuth2UserService`에서 Google의 `providerId`를 `"id"` 대신 `"sub"`로 추출하도록 수정 — 여전히 실패
- 근본 원인 발견: Google은 `openid` scope 사용 → OIDC 플로우 → `OidcUserService` 호출됨, `SecurityConfig`에는 `userService()`만 등록되어 `CustomOAuth2UserService`가 Google 로그인 시 호출되지 않음
- `CustomOidcUserService.java` 신규 생성 (`OidcUserService` 상속, `oidcUser.getSubject()`로 providerId 추출)
- `SecurityConfig`에 `.oidcUserService(customOidcUserService)` 등록
