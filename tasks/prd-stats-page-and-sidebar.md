# PRD: 통계 페이지 & 글로벌 사이드바 네비게이션

## 개요

링크별 클릭 통계를 Chart.js 차트로 시각화하는 전용 통계 페이지를 추가하고, 현재 상단 네비게이션만 있는 레이아웃을 글로벌 좌측 사이드바 네비게이션으로 교체한다. 현재 백엔드에는 `StatsV1` 데이터(총 클릭수, 고유 클릭수, 일별 클릭수)를 제공하는 `GET /api/links/{linkId}/stats` API가 이미 존재하지만, 이를 시각화하는 UI 페이지가 없다. 대시보드와 링크 상세 페이지는 사이드바 없이 단순한 상단 네비게이션 바만 사용하고 있다.

## 목표

- 링크별 전용 통계 페이지(`/app/links/{id}/stats`)를 추가하여 Chart.js 차트로 기본 클릭 분석 데이터 시각화
- 현재 상단 네비게이션 레이아웃을 모든 인증 페이지에 적용되는 좌측 사이드바로 교체 (Dashboard, My Links, Stats, Settings 플레이스홀더)
- 통계 날짜 범위 프리셋 필터 지원 (오늘, 7일, 30일, 전체)
- 기존 `AnalyticsService.getStatsV1()` 백엔드 재사용 -- 새로운 API 엔드포인트 불필요

## 사용자 스토리

### US-001: 글로벌 사이드바 네비게이션 레이아웃 추가
**설명:** 사용자로서, 모든 인증 페이지에 좌측 사이드바가 있어 Dashboard, My Links 등 섹션 간 빠르게 이동할 수 있기를 원한다.

**인수 조건:**
- [ ] 사이드바가 모든 인증 페이지에 표시됨: 대시보드, 링크 생성, 링크 상세, 통계
- [ ] 사이드바에 네비게이션 항목 포함: Dashboard (`/app/dashboard`), My Links (현재는 대시보드와 동일), Settings 플레이스홀더 (비활성, 회색 처리)
- [ ] 현재 페이지가 사이드바에서 시각적으로 강조 표시됨
- [ ] 사이드바 상단에 앱 로고/이름, 하단에 사용자 이름 + 로그아웃 버튼 포함
- [ ] 인증 페이지에서 기존 상단 네비게이션 바 제거 (랜딩 페이지와 로그인 페이지는 유지)
- [ ] 메인 컨텐츠 영역이 사이드바를 수용하도록 우측으로 이동 (사이드바 너비 ~240px)
- [ ] 반응형: 화면 너비 < 768px에서 사이드바 숨김 또는 축소
- [ ] 빌드 통과 (`./gradlew compileJava`)

### US-002: 링크별 통계 페이지 생성
**설명:** 사용자로서, 특정 링크의 클릭 통계를 전용 페이지에서 확인하여 단축 링크의 성과를 파악하고 싶다.

**인수 조건:**
- [ ] `/app/links/{id}/stats` 경로로 접근 가능한 새 페이지
- [ ] 총 클릭수(큰 숫자), 고유 클릭수(큰 숫자), Chart.js 라인 차트("일별 클릭 추이") 표시
- [ ] 차트 X축에 날짜 라벨, Y축에 클릭수 표시
- [ ] 링크 상세 페이지(`/app/links/{id}`)에 "통계 보기" 버튼/링크 추가하여 통계 페이지로 이동
- [ ] 통계 페이지 상단에 링크 제목과 단축 URL 표시 (컨텍스트 제공)
- [ ] 통계 페이지에 사이드바 네비게이션 포함 (US-001)
- [ ] 빌드 통과

### US-003: 통계 페이지에 날짜 범위 프리셋 필터 추가
**설명:** 사용자로서, 프리셋 날짜 범위로 통계를 필터링하여 최근 또는 과거 성과에 집중하고 싶다.

**인수 조건:**
- [ ] 차트 위에 4개 프리셋 버튼 표시: "오늘", "7일", "30일", "전체"
- [ ] 페이지 로드 시 기본 선택은 "30일"
- [ ] 프리셋 클릭 시 `from`/`to` 쿼리 파라미터로 페이지 리로드
- [ ] 선택된 프리셋 버튼이 시각적으로 강조 (활성 상태)
- [ ] "전체"는 링크의 `createdAt` 날짜를 `from` 파라미터로 사용
- [ ] 통계 숫자(총 클릭수, 고유 클릭수)도 선택한 날짜 범위에 맞게 업데이트
- [ ] 빌드 통과

### US-004: 통계 뷰 컨트롤러 엔드포인트 생성
**설명:** 개발자로서, 통계 데이터를 조회하고 Thymeleaf 통계 템플릿을 렌더링하는 서버사이드 컨트롤러 엔드포인트가 필요하다.

**인수 조건:**
- [ ] 뷰 컨트롤러에 `GET /app/links/{id}/stats` 엔드포인트 추가
- [ ] 선택적 쿼리 파라미터 `from`, `to` (LocalDate) 수용, 기본값은 최근 30일
- [ ] 링크 소유권 검증 (현재 사용자가 링크 소유자여야 함)
- [ ] `AnalyticsService.getStatsV1(linkId, from, to)` 호출 후 데이터를 템플릿에 전달
- [ ] 링크 엔티티, 통계 데이터, 활성 날짜 프리셋 이름을 템플릿 모델에 전달
- [ ] 링크를 찾을 수 없거나 소유자가 아닌 경우 404 반환
- [ ] 빌드 통과

### US-005: Thymeleaf 사이드바 프래그먼트로 리팩토링
**설명:** 개발자로서, 모든 인증 페이지가 코드 중복 없이 동일한 네비게이션을 공유하도록 사이드바 레이아웃의 공유 Thymeleaf 프래그먼트가 필요하다.

**인수 조건:**
- [ ] 사이드바 HTML을 포함하는 레이아웃 프래그먼트 생성 (예: `templates/fragments/sidebar.html`)
- [ ] 대시보드, 링크 생성, 링크 상세, 통계 페이지 모두 이 프래그먼트 포함
- [ ] 사이드바 CSS 공유 (프래그먼트 또는 공유 스타일 블록에 포함)
- [ ] 한 페이지에서 사이드바 프래그먼트를 제거해도 다른 페이지에 영향 없음
- [ ] 빌드 통과

## 기능 요구사항

- FR-1: 네비게이션 링크(Dashboard, My Links, Settings 플레이스홀더)를 포함하는 공유 Thymeleaf 사이드바 프래그먼트 생성
- FR-2: 모든 인증 페이지에 사이드바 레이아웃 적용: `dashboard.html`, `link-create.html`, `link-detail.html`, 새 `stats.html`
- FR-3: 인증 페이지에서 상단 네비게이션 바 제거; 랜딩 페이지와 로그인 페이지에만 유지
- FR-4: `StatsV1` 데이터를 조회하고 통계 템플릿을 렌더링하는 `GET /app/links/{id}/stats` 컨트롤러 엔드포인트 추가
- FR-5: 통계 페이지에 총 클릭수와 고유 클릭수를 큰 카드로 표시
- FR-6: 통계 페이지에 선택한 날짜 범위의 일별 클릭수를 Chart.js 라인 차트로 렌더링
- FR-7: 4개 날짜 범위 프리셋 버튼 제공: 오늘, 7일, 30일, 전체 (기본값: 30일)
- FR-8: 날짜 범위 프리셋이 `?from=YYYY-MM-DD&to=YYYY-MM-DD` 쿼리 파라미터로 페이지 리로드
- FR-9: 링크 상세 페이지(`/app/links/{id}`)에 `/app/links/{id}/stats`로 이동하는 "통계 보기" 링크 추가
- FR-10: 사이드바가 URL 경로 기반으로 현재 활성 네비게이션 항목 강조

## 범위 외 (Non-Goals)

- StatsV2 데이터 (리퍼러, 디바이스 분석, 브라우저 분석) 미포함 -- 기본 통계만
- 실시간 / 자동 새로고침 차트 없음
- 내보내기 기능 (CSV, PDF) 없음
- 전체 링크 집계 통계 대시보드 없음 (링크별 통계만)
- Settings 페이지 구현 없음 (사이드바 링크는 플레이스홀더)
- 다크 모드 / 테마 전환 없음
- 모바일 햄버거 메뉴 없음 (작은 화면에서 단순 숨김으로 충분)

## 디자인 고려사항

- **사이드바 너비:** ~240px, 고정 위치, 전체 높이, 흰색/밝은 배경에 우측 미세한 테두리
- **색상 체계:** 기존 `#4285F4` 파란색을 프라이머리 액센트 컬러로 계속 사용
- **Chart.js CDN:** `<script src="https://cdn.jsdelivr.net/npm/chart.js">`로 로드 -- npm/번들러 설정 불필요
- **통계 카드:** 차트 위에 두 개의 가로 카드 (총 클릭수, 고유 클릭수) -- 큰 숫자 표시
- **프리셋 버튼:** 가로 행의 필/탭 버튼, 활성 버튼은 프라이머리 색상으로 채움
- **수정할 기존 템플릿:** `dashboard.html`, `link-create.html`, `link-detail.html`
- **새 템플릿:** `stats.html`, `fragments/sidebar.html`

## 기술 고려사항

- **새로운 백엔드 API 불필요.** 기존 `AnalyticsService.getStatsV1()`이 `StatsV1(totalClicks, uniqueClicks, clicksByDate)`를 반환하므로 필요한 데이터가 모두 있음. 뷰 컨트롤러에서 직접 호출.
- **Chart.js 데이터 전달:** Thymeleaf의 `th:inline="javascript"`를 사용하여 `clicksByDate` 리스트 (날짜 라벨 + 카운트)를 `<script>` 블록에 JSON으로 주입.
- **날짜 범위 로직:** 컨트롤러가 프리셋 이름 또는 명시적 파라미터에서 `from`/`to` 계산. "전체"는 `link.getCreatedAt()`을 `from`으로 사용.
- **소유권 검사:** `DashboardViewController.linkDetail()`의 기존 패턴 재사용 -- `linkService.findByIdAndUserId(id, userId)`.
- **사이드바 프래그먼트:** `th:fragment="sidebar(activePage)"` 파라미터로 강조 네비게이션 항목 제어.

## 성공 지표

- 클릭 데이터가 있는 링크에 대해 통계 페이지가 차트를 올바르게 로드 및 렌더링
- 클릭이 0인 링크에 대해 적절한 빈 상태 표시
- 사이드바 네비게이션이 모든 인증 페이지에서 레이아웃 깨짐 없이 동작
- 날짜 범위 프리셋이 표시 데이터를 올바르게 필터링
